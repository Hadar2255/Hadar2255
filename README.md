# 🤖 ויקטור — בוט וואטסאפ לקבוצה

ויקטור הוא בוט חכם לקבוצות וואטסאפ שזוכר עבור החברים בקבוצה:

- 🛒 **רשימות קניות** — הוסף, הצג, נקה
- ✅ **משימות** — פתוחות, מבוצעות, עם תאריכי יעד
- 📅 **לוח זמנים** — אירועים, פגישות, סופ"ש

הבוט משתמש ב־**Gemini** (Google) להבנת שפה טבעית בעברית, ב־**Baileys** לחיבור לוואטסאפ, וב־**SQLite** לשמירת המידע. נתונים נשמרים בנפרד לכל קבוצה.

## איך זה עובד

פנו אל הבוט בקבוצה לפי השם שלו (ברירת מחדל: `ויקטור`):

```
ויקטור תוסיף לקניות חלב, ביצים ולחם
ויקטור מה ברשימת הקניות?
ויקטור תזכיר לי לקבוע תור לרופא ביום שלישי
ויקטור מה המשימות שלנו?
ויקטור קבענו ארוחת ערב ביום שישי ב־20:00 אצל אמא
ויקטור מה הלו"ז לסופש?
ויקטור סיימתי לקבוע תור לרופא
ויקטור תמחק מהקניות ביצים
```

הבוט יחזיר הודעה מעוצבת יפה עם הרשימה.

## דרישות

- Node.js 18+
- מפתח API ל־Google Gemini (חינמי) — [צור כאן](https://aistudio.google.com/apikey)
- מספר טלפון נפרד או מקושר לוואטסאפ (בוט מתחבר כמו "מכשיר מקושר")

## התקנה

```bash
git clone https://github.com/Hadar2255/Hadar2255.git
cd Hadar2255
npm install

cp .env.example .env
# ערוך את .env והכנס את GEMINI_API_KEY שלך
```

## הרצה

```bash
npm start
```

בריצה הראשונה תקבל קוד QR בטרמינל. סרוק אותו מהוואטסאפ:

> וואטסאפ → הגדרות → מכשירים מקושרים → קישור מכשיר

לאחר הסריקה הבוט מחובר. הוסף אותו לקבוצה (דרך הטלפון שמולו אתה מקושר) והתחל לדבר איתו.

## הוספת הבוט לקבוצה

הבוט מתחבר עם מספר הטלפון שסרק את ה־QR. כדי לצרף אותו לקבוצה:

1. בקבוצה, פתח **פרטי הקבוצה → הוסף משתתף**.
2. הוסף את המספר של הטלפון שמשמש את הבוט.
3. כתוב בקבוצה: `ויקטור עזרה` כדי לקבל מדריך שימוש.

## הגדרה (`.env`)

| משתנה            | ברירת מחדל          | הסבר                                            |
| ---------------- | ------------------- | ----------------------------------------------- |
| `GEMINI_API_KEY` | _חובה_              | מפתח API מ־Google AI Studio                     |
| `GEMINI_MODEL`   | `gemini-2.5-flash`  | מודל Gemini (flash מהיר וזול, pro חכם יותר)     |
| `BOT_NAME`       | `ויקטור`            | השם שאליו פונים בקבוצה                          |
| `DB_PATH`        | `./data/bot.db`     | מיקום מסד הנתונים                               |

## מבנה הקוד

```
src/
├── index.js      — חיבור Baileys, QR, ניהול חיבור
├── bot.js        — זיהוי פנייה לבוט והפעלת הפעולה
├── ai.js         — שליחה ל־Gemini ופירוק כוונה (intent)
├── db.js         — שכבת SQLite (פריטים מסוגים שונים)
└── formatter.js  — עיצוב הודעות וואטסאפ עם אימוג'י
```

## הערות

- **בידוד**: כל קבוצה רואה רק את הנתונים שלה (`group_jid` הוא המפתח).
- **חיבור מתמיד**: הבוט מתחבר מחדש אוטומטית בנפילת רשת.
- **התנתקות**: כדי לאפס חיבור, מחק את תיקיית `auth_info/` והרץ שוב.

## 🔐 אבטחה ופרטיות

### איפה המידע ומה יוצא החוצה

```
WhatsApp (E2E) → המחשב שלך (Baileys) → SQLite מקומי (data/bot.db)
                                       │
                          (כשפונים לבוט) → Google Gemini API
```

**מקומית בלבד:**
- כל ההודעות בקבוצה (עד 500 אחרונות לקבוצה) — בקובץ `data/bot.db`
- רשימות (קניות / משימות / לו"ז) — אותו קובץ
- אישורי וואטסאפ — בתיקיית `auth_info/`
- מפתח Gemini API — ב־`.env`

**יוצא החוצה רק כשפונים לבוט במפורש:**
- ההודעה הספציפית של הפונה
- 30 הודעות אחרונות של הקבוצה (כקונטקסט להבנה) — *אלא אם מצב פרטי מופעל*
- מצב הרשימות הנוכחי
- היעד: Google Gemini API דרך HTTPS

### הגנות פעילות

1. **הצפנת AES-256-GCM** של תוכן ההודעות והרשימות בקובץ DB. כדי להפעיל, הגדר `BOT_ENCRYPTION_KEY` ב־`.env` (סיסמה חזקה של 12+ תווים). בלי המפתח, מי שיש לו גישה לקובץ DB *לא יכול* לקרוא תכנים.
2. **הצפנה בתעבורה**: WhatsApp E2E + HTTPS ל־Gemini.
3. **`.gitignore`** מונע העלאה בטעות של `.env`, `data/`, `auth_info/`.
4. **בידוד לפי קבוצה**: בוט בקבוצה אחת לא נחשף לנתונים של קבוצה אחרת.

### מצב פרטי (Private Mode)

הגדר `BOT_PRIVATE_MODE=1` ב־`.env`. במצב זה:
- ל־Gemini נשלחת **רק ההודעה הספציפית שפנתה לבוט**.
- 30 ההודעות האחרונות של הקבוצה **לא נשלחות**.
- הבוט מאבד יכולת "להעלות" דברים שנאמרו אגב אורחא בקבוצה.
- שאר התכונות עובדות רגיל.

### פקודות פרטיות

מתוך הקבוצה אפשר לומר:
- `ויקטור סטטוס אבטחה` — בודק אם הצפנה ומצב פרטי פעילים.
- `ויקטור שכח את ההודעות האחרונות` — מוחק 5 דקות של היסטוריה.
- `ויקטור שכח את ה־30 דקות האחרונות` — חלון מותאם.
- `ויקטור שכח הכל` — מוחק את כל ההיסטוריה של הקבוצה (הרשימות נשארות).

### מה *לא* מובטח (חשוב להבין)

- **Gemini בתוכנית החינמית**: גוגל יכולה להשתמש בקלט לשיפור מודלים. בתוכנית בתשלום (Tier 1+) הם לא משתמשים. ראה https://ai.google.dev/gemini-api/terms.
- **כשהבוט פעיל**: בזיכרון התהליך, ההודעות מפוענחות לזמן עיבוד. הצפנה היא ל־*storage*, לא ל־*runtime*.
- **גישה למחשב = גישה לכל**: מי שמתחבר למחשב שלך יכול לקרוא את `.env` ולפענח את ה־DB.
- **חברי הקבוצה רואים תשובות**: זה ברור, אבל שווה להזכיר.
- **מטא-דאטה ב־WhatsApp**: WhatsApp רואה מי משוחח עם מי, מתי וכמה — לא תוכן.

### המלצות מעשיות

1. הגדר `BOT_ENCRYPTION_KEY` עם סיסמה חזקה ושמור אותה במנהל סיסמאות.
2. אל תשתף צילומי מסך של הטרמינל בזמן הרצת הבוט (יכולים לחשוף מפתחות).
3. הגבל הרשאות קובץ:
   - **Mac/Linux**: `chmod 600 .env && chmod 700 data/ auth_info/`
   - **Windows**: דרך File Properties → Security
4. אם חשדת שמפתח Gemini דלף — בטל אותו מיד ב־https://aistudio.google.com/apikey.
5. אם חשדת שאישורי וואטסאפ דלפו — מחק `auth_info/` והרץ שוב.
6. בקבוצות רגישות במיוחד — הפעל `BOT_PRIVATE_MODE=1`.

## פתרון בעיות

- **"Missing GEMINI_API_KEY"** — הכנס את המפתח ב־`.env`.
- **לא רואה QR** — נסה להגדיל את חלון הטרמינל; הקוד צריך להיות מרובע.
- **הבוט לא מגיב בקבוצה** — ודא שאתה מתחיל את ההודעה במילה "ויקטור" (או בשם שהגדרת ב־`BOT_NAME`).
- **`npm install` נכשל** — ודא שהגרסה של Node היא 18 ומעלה (`node --version`). הספרייה משתמשת ב־SQLite מבוסס WASM, אז אין צורך במהדר נייטיב.
